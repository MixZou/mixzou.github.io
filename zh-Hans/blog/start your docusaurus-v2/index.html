How To Deploy Your Site

2022年5月18日 · 4 分钟阅读
MixZou
嘎呜

Welcome to this blog. 本篇文章将会介绍如何将Docusaurus部署到GitHub Page上。

Docusaurus是什么​

docusaurus是基于React框架的静态站点生成器，通俗来讲，可以快速生成一个文档网站，可用于打造自己的博客等。

安装​

  1. 由于本质是一组npm包的集合，所以需要提前配置好node.js环境
  2. 选择好你准备的项目路径，在此处运行npx create-docusaurus@latest my-website classic，其中my-website是你项目的名字可以自定义，classic是经典的生成模板
  3. 进入my-website路径，使用VS Code或者其它你使用的编译器打开项目，新建一个终端，输入命令npm run start在本地运行项目查看效果，在命令行使用control+c或者crtrl + c退出
  4. 输入命令npm run build构建项目，生成build目录

部署​

众所周知，写好了网站，自然想让它上线，可以让互联网访问。为此，你可以购买服务器和域名尽情客制化你的网站，当然也可以让你自己的电脑做服务器，但是不推荐这一方法。

GitHub Page提供免费的部署功能，以下是前置条件和介绍。

  • 你有一个GitHub账号
  • 你已经配置好本地的Git，例如账户和秘钥
  • 你会基本的Git操作
  • 你的电脑已经配置好yarn环境

GitHub仓库​

  1. 新建GitHub仓库，名称为username.github.io，username是你的账户名称
  2. 不用自动生成README.md
  3. 仓库设置为公开访问
  4. 根据仓库生成页面的提示，将本地已存在的website， pull到这个仓库，git init那些我就不说了，需要注意的是请在第一次pull之前创建分支gh-pages而不是master，因为Docusaurus在GitHub Page的默认部署分支是这个，特别是.io结尾的
  5. 在仓库的setting中，左侧边栏page页面，设置默认分支为gh-pages

本地website​

找到docusaurus.config.js，进行部署的配置

const config = {
  title: 'Abbys Of Code',   //网站的标题
  tagline: '一只小菜鸡的成长记录',    //网站首页的描述
  url: 'https://github.com/MixZou/mixzou.github.io',    //你的仓库网址
  baseUrl: '/', //必须带'/'结尾
  onBrokenLinks: 'throw',
  onBrokenMarkdownLinks: 'warn',
  favicon: 'img/favicon.ico',

  // GitHub pages deployment config.
  // If you aren't using GitHub pages, you don't need these.
  organizationName: 'mixzou', // 你的 GitHub 用户名
  projectName: 'mixzou.github.io', // Usually your repo name，也是部署后的访问网址
  deploymentBranch: 'gh-pages', //设置默认分支

在终端输入命令yarn deploy，项目将自动进行build，然后部署到你的远程仓库。

成功后终端会显示Website is live at "https://mixzou.github.io/". ✨ Done in 56.43s.

一般过一两分钟，网站就会变化啦，记得是https://+reponame，不是GitHub仓库地址唷。
